Live Studio — Domain Glossary

The canonical vocabulary for Live Studio v1. Use these terms in code, docs, and conversation. Source: CONTEXT.md

People & Identity

Host
The person who owns a Room, joins with their own camera and microphone, invites Guests, controls what goes live, removes Guests, and starts or stops the broadcast.
Author, creator, room owner
Account
A sign-in identity required for a Host to create and return to Rooms. Guests do not need Accounts in v1.
User profile, member, workspace
Guest
An invited contributor in a Room who can share their own camera and microphone. Guests do not control invitations, broadcast state, or other participants. A Guest can be removed, force-muted, or forced camera-off by the Host.
Participant, speaker, attendee
Display Name
The name a person types before entering a Room. Appears inside the Room and in the Composed Room Feed. Limited to 50 characters, plain text.
Username, profile name, handle
Audience
People who watch the broadcast outside the Room on YouTube. The Audience is not part of the Room and does not join as Guests.
Viewer, attendee, participant

The Room

Room
A reusable private live space created by one Host. Up to three Guests can be invited. While Broadcasting, everyone in the Room is visible to the Audience.
Studio, meeting, session
Room Title
A Host-facing name for a reusable Room. Helps the Host manage Rooms. Not part of the Composed Room Feed in v1.
Show title, event name, stream title
Room Full
The state shown when an invited person tries to enter a Room that already has the Host and three Guests. No waiting queue in v1.
Waiting room, queue, overflow
Guest Invite
A hard-to-guess link (≥128-bit URL-safe token) created by the Host that grants a Guest entry into a Room. Can be revoked or regenerated.
Meeting link, access code, participant URL
Join Check
The pre-entry step: type Display Name, grant camera/mic permissions, choose devices, set initial state, preview yourself.
Lobby, setup screen, waiting room
Room Chat
Private text conversation between Host and Guests. Not visible to the Audience. Not persisted in v1.
Audience chat, comments, messages
Backstage
The state of a Room where participants can see and hear each other, but the Audience cannot watch. Default state; a Room returns to Backstage when a Broadcast ends.
Preview, waiting room, pre-live

Broadcasting

Broadcast
One live run of a reusable Room sent to one Broadcast Destination. A Room can have many Broadcasts. Each uses its own ephemeral stream credentials.
Stream, event, session
Broadcasting
The Room state with an active Broadcast: everyone in the Room is visible to the Audience through a Broadcast Destination.
Live, streaming, published
Broadcast State
Room-visible state: Backstage → Countdown → Broadcasting → Ended or Failed. Visible to everyone in the Room.
Room state, live status, stream state
Broadcast Countdown
A short (5-second) countdown shown to Host and Guests before Broadcasting starts. Host can cancel before it completes. Added in Milestone 3.
Timer, pre-roll, slate
Broadcast Health
Host-visible operational condition: connecting, broadcasting, degraded, failed, or ended. Guests see only Broadcast State, not health details.
Status, diagnostics, stream state
Broadcast Destination
An external platform receiving the Composed Room Feed. V1 supports YouTube, keeping the concept compatible with RTMP destinations.
Channel, output, stream target

Media & Composition

Composed Room Feed
A single audio/video feed of everyone in the Room arranged as one broadcast output. Targets 720p at 30fps. Includes participant name labels and camera-off placeholders.
Separate participant feeds, individual sources, raw tracks
Broadcast Preview
The Room-visible preview of the Composed Room Feed. While Backstage: shows what would go live. While Broadcasting: shows what is being sent. Host can toggle; Guests always see it during Broadcasting.
Program monitor, output preview, canvas preview
Screen Share
A shared display, window, or browser tab contributed by the Host. One active at a time. When active, it becomes the primary visual in the Composed Room Feed with participant cameras as smaller tiles.
Presentation, slides, desktop source
Broadcast Bridge
Server-side service carrying the Composed Room Feed to the Broadcast Destination. Implemented with GStreamer (WHIP ingest → H.264/AAC → RTMP sink). Runs as a separate Podman Compose service.
OBS, stream client, relay

Technical Terms

Supported Browser
Chromium-based browser (Chrome or Edge) for Hosts. Chrome on Android for mobile Guests. iOS Safari is out of scope for v1.
Safari, Firefox, unsupported browser

Flagged — Not In V1

Stage
Host-selected subset of participants visible to Audience. In v1, everyone in the Room is visible.
Producer-only Host
A Host who controls but is not visible. In v1, a Host is always a visible participant.
Guest Screen Share
Only the Host can Screen Share in v1.
Brand Kit
Logos, banners, custom backgrounds, themes, lower-thirds. Outside v1 scope.
Scheduled Broadcast
Host manually starts a Broadcast in v1. No scheduling.
