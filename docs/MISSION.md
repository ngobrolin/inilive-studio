# Mission

## Why am I learning this?

I'm about to build Live Studio — a browser-based live production studio where a Host brings Guests into a Room and broadcasts the conversation to YouTube. I need to deeply understand the project plan, architecture, and domain before writing any code.

## What does success look like?

I can confidently start Milestone 1 (Local Room Prototype) knowing:

- The full product domain and ubiquitous language
- How WebRTC, SFUs, and browser media APIs work
- How canvas composition and `captureStream()` produce the Composed Room Feed
- The SvelteKit architecture and how it maps to the plan
- What each milestone proves and why they're ordered this way
- The key risks and where to expect difficulty
